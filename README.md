## 📖 项目简介
这是一个由全栈开发者构建的 Python 全阶段练习库。项目采用分层设计，提供从基础语法到复杂系统的完整学习路径。

本仓库包含 **570+个精心设计的练习**，覆盖 Python 的核心概念和实际应用场景。每个文件都是独立可运行的，包含**标准实现**与**详细解析**双版本，并配套完整的 pytest 测试用例，帮助不同水平的学习者快速掌握 Python 编程。

## 🎮 难度分级体系
本项目采用渐进式学习机制，共分为 9 个等级，按难度递增排列：

| 等级 | 前缀 | 名称 | 描述 | 关键技能点 |
| :--- | :--- | :--- | :--- | :--- |
| **Lv.1** | **B** | **Basics (基础)** | 起点。Hello World，变量，print。 | 基础语法，变量类型，输入输出 |
| **Lv.2** | **S** | **Simple (简单)** | 上手。写一些只有十几行的逻辑。 | 条件判断，循环，基础算法逻辑 |
| **Lv.3** | **N** | **Normal (普通)** | 日常。开始用函数封装，处理文件。 | 函数，文件读写，模块化思维 |
| **Lv.4** | **H** | **Hard (困难)** | 爬坡。引入面向对象（OOP）。 | 类与对象，继承，封装，多态 |
| **Lv.5** | **C** | **Complex (复杂)** | 架构。多模块协作，异常处理。 | 异常捕获，多文件协作，数据库连接 |
| **Lv.6** | **A** | **Advanced (进阶)** | 深入。使用外部库，网络请求。 | `requests`, `pandas`, GUI, 数据可视化 |
| **Lv.7** | **E** | **Expert (专家)** | 专业。代码规范，完善工具。 | CLI工具，代码规范，健壮性设计 |
| **Lv.8** | **M** | **Master (大师)** | 掌控。从零构建完整系统。 | Web后端，数据分析工具，系统架构 |
| **Lv.9** | **N** | **Nightmare (噩梦)** | 终极挑战。开发完整 2D 游戏。 | 游戏循环，物理碰撞，状态机，资源调度 |

## 📁 文件命名规范
为了方便检索，项目采用 **"难度前缀 + 三位编号 + 功能描述"** 的命名方式：

*   **格式**：`{难度前缀}{三位编号}_{功能描述}.py`
*   **示例**：
    *   `B001_hello_world.py` (基础阶段：Hello World)
    *   `S009_guess_number.py` (简单阶段：猜数字游戏)
    *   `N010_simple_calculator.py` (普通阶段：简易计算器)
    *   `N016_terminal_rpg.py` (噩梦阶段：文字RPG游戏)

## 📝 代码规范与格式
每个练习文件都遵循统一的结构：

### 1. 文件头部
包含题目描述和制作提示，帮助理解练习目标。

### 2. 代码主体

模拟真实控制台输出，直观展示程序行为。

## 📋 练习列表

### Basics (基础) - 90个练习

涵盖数据类型、运算符、控制流、函数、类、模块、异常处理、文件操作、正则表达式、装饰器、生成器等核心基础知识点。

### Simple (简单) - 136个练习

涵盖基础算法、排序、查找、递归、回溯、链表、二叉树、动态规划等进阶编程概念。

### Normal (普通) - 50个练习

涵盖函数封装、文件处理、模块化设计、小型应用系统开发等实用技能。

### Hard (困难) - 50个练习

涵盖面向对象编程、类设计、继承封装多态、业务系统开发等高级OOP技能。

### Complex (复杂) - 50个练习

涵盖设计模式、多模块协作、异常处理、数据库连接等架构级编程技能。

### Advanced (进阶) - 50个练习

涵盖异步编程、依赖注入、ORM、缓存、路由、验证框架等高级开发技能。

### Expert (专家) - 50个练习

涵盖元编程、AST解析、代码生成、AOP框架、插件架构等专家级技能。

### Master (大师) - 50个练习

涵盖事件溯源、CQRS、Saga模式、熔断器、分布式系统等架构师级技能。

### Nightmare (噩梦) - 50个练习

## 🛠️ 如何使用本仓库

1.  **按需学习**
    根据你的当前水平选择合适的难度级别开始学习。建议从 Basics 模块开始，循序渐进。

2.  **先想后看**
    强烈建议先尝试自己编写，卡住时再查看标准实现，理解后再阅读详细解析加深印象。

3.  **动手实践 + 测试验证**
    每个练习都是可运行的，建议实际运行代码并尝试修改。同时运行测试用例验证代码正确性。

    ```bash
    # 运行单个练习
    python basics/B001_hello_world.py

    # 运行所有测试
    pytest tests/test_basics.py -v
    ```

4.  **扩展挑战**
    尝试修改代码实现更多功能，或添加新的测试用例。
