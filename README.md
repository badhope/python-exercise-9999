## 📖 项目简介
这是一个由全栈开发者从零构建的 Python 全阶段练习库。项目不只是简单的代码堆砌，而是一条**从脚本小子到架构师**的完整进阶路线。
本仓库旨在通过 **1000+ 个独立文件**，覆盖 Python 的每一个细节。无论你是刚学会 `print` 的新手，还是想要通过代码构建复杂系统的开发者，这里都有适合你的挑战。每个文件都是独立可运行的，且包含**普通答案**与**保姆级解析**双版本，确保你不仅会写，更懂为什么这么写。

## 🎮 难度分级体系
本项目采用游戏化的进阶机制，共分为 9 个等级。你需要通关前一阶段，才能更好挑战下一阶段。
| 等级 | 名称 | 描述 | 关键技能点 |
| :--- | :--- | :--- | :--- |
| **Lv.1** | **Basics (基础)** | 起点。Hello World，变量，print。 | 基础语法，变量类型，输入输出 |
| **Lv.2** | **Simple (简单)** | 上手。写一些只有十几行的逻辑。 | 条件判断，循环，基础算法逻辑 |
| **Lv.3** | **Normal (普通)** | 日常。开始用函数封装，处理文件。 | 函数，文件读写，模块化思维 |
| **Lv.4** | **Hard (困难)** | 爬坡。引入面向对象（OOP）。 | 类与对象，继承，封装，多态 |
| **Lv.5** | **Complex (复杂)** | 架构。多模块协作，异常处理。 | 异常捕获，多文件协作，数据库连接 |
| **Lv.6** | **Advanced (进阶)** | 深入。使用外部库，网络请求。 | `requests`, `pandas`, GUI, 数据可视化 |
| **Lv.7** | **Expert (专家)** | 专业。代码规范，完善工具。 | CLI工具，代码规范，健壮性设计 |
| **Lv.8** | **Master (大师)** | 掌控。从零构建完整系统。 | Web后端，数据分析工具，系统架构 |
| **Lv.9** | **Nightmare (噩梦)** | 终极挑战。开发完整 2D 游戏。 | 游戏循环，物理碰撞，状态机，资源调度 |

## 📁 文件命名规范
为了方便检索，项目采用 **“阶段前缀 + 描述性命名”** 的方式，让你一眼就能看出文件是做什么的。
*   **格式**：`{阶段序号}_{简要功能描述}.py`
*   **示例**：
    *   `01_hello_world.py` (基础阶段：Hello World)
    *   `simple_guess_number.py` (简单阶段：猜数字游戏)
    *   `normal_simple_calculator.py` (普通阶段：简易计算器)
    *   `nightmare_terminal_rpg.py` (噩梦阶段：文字RPG游戏)

## 📝 代码规范与格式
这是本项目的核心特色。每一个练习文件都严格遵守以下结构，确保学习体验的一致性。
### 1. 文件头部
包含编码声明、题目描述，从第二阶段开始包含**制作提示**。
### 2. 代码主体
*   **普通答案**：最简洁、最实用的实现方式，供老手快速参考。
*   **运行效果**：模拟真实控制台输出，直观展示程序行为（从第二阶段开始）。
*   **详细解析版**：每一行代码上方都有注释，解释其作用，适合新手逐行研读。
### 3. 底部说明
*   **大白话解释**：用生活中的打比方（如“变量是盒子”、“函数是工具箱”）解释核心逻辑，拒绝晦涩术语。
*   **扩展语句**：提供进阶思路，引导读者修改代码，举一反三。
### 4. 题目示例 (ACM风格)
从第三阶段开始，题目描述中将包含明确的 **输入示例** 和 **输出示例**，规范化输入输出格式。

## 🛠️ 如何使用本仓库
1.  ** Clone 仓库 **
    ```bash
    git clone https://github.com/badhope/python-exercises-1000.git
    ```
2.  **按需学习**
    如果你是初学者，请从 `01_` 开头的文件看起；如果你有一定基础，可以直接跳到 `Normal` 或 `Hard` 阶段。
3.  **先想后看**
    强烈建议先尝试自己编写，卡住时再查看“普通答案”，理解后再阅读“详细解析版”加深印象。
